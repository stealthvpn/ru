Что такое X-UI и зачем он нужен

X-UI (активная форка — 3x-ui) — это веб-панель с открытым исходным кодом для управления Xray-core — современным прокси-сервером, который поддерживает протоколы VLESS, VMess, Trojan, Shadowsocks и другие[reference:3][reference:4]. Панель предоставляет удобный графический интерфейс, который позволяет:

  • Создавать и управлять входящими подключениями (inbounds) для разных протоколов.
  • Добавлять пользователей с ограничениями по трафику и времени действия.
  • Настраивать транспортные протоколы: WebSocket, gRPC, XHTTP и другие.
  • Мониторить статистику трафика в реальном времени.
  • Автоматически обновлять Xray-core и сертификаты[reference:5].

Благодаря X-UI даже новичок может развернуть профессиональный VPN-сервер за 10–15 минут, не вводя сложных команд вручную.

Подготовка сервера

Перед установкой X-UI необходимо подготовить сервер. Мы будем использовать Ubuntu 22.04 LTS — это рекомендуемая ОС для стабильной работы[reference:6][reference:7].

1. Подключитесь к серверу по SSH:
ssh root@ваш_ip_сервера
2. Обновите систему:
sudo apt update && sudo apt upgrade -y
3. Установите необходимые пакеты:
sudo apt install -y wget curl tar tzdata cron ca-certificates nftables[reference:8]
4. Включите BBR для ускорения TCP:
echo "net.core.default_qdisc=fq" >> /etc/sysctl.conf
echo "net.ipv4.tcp_congestion_control=bbr" >> /etc/sysctl.conf
sysctl -p[reference:9]
5. Настройте файрвол (UFW) — откройте порты:
sudo ufw allow 22/tcp (SSH)
sudo ufw allow 443/tcp (для Reality)
sudo ufw allow 2053/tcp (порт панели, можно изменить)
sudo ufw enable[reference:10]

🔧 Команда установки:

После завершения установки вы увидите сообщение с адресом панели, именем пользователя и паролем. Обязательно сохраните эти данные![reference:13]

Настройка VLESS Reality с транспортом XHTTP

Теперь создадим входящее подключение с протоколом VLESS, технологией Reality и транспортом XHTTP — это современная связка, которая обеспечивает максимальную скрытность трафика[reference:17][reference:18].

1. В панели перейдите в раздел Inbounds и нажмите Add Inbound.
2. Заполните параметры:
  • Protocol: VLESS
  • Port: 443 (или любой другой, но 443 рекомендуется для маскировки)
  • Flow: xtls-rprx-vision (рекомендуется для Reality)[reference:19]
  • Transport: XHTTP[reference:20]
3. Настройте Reality:
  • Включите Reality в настройках.
  • Укажите Server Name (SNI) — например, www.google.com или www.cloudflare.com.
  • Остальные параметры оставьте по умолчанию или сгенерируйте ключи автоматически.
4. Настройте XHTTP:
  • В поле Host укажите ваш домен или IP[reference:21].
  • В поле Path можно указать любой путь, например /.
5. Нажмите Add — подключение создано!

После создания inbound-а вы увидите ссылку для подключения (в формате vless://...). Её можно скопировать и использовать в клиентских приложениях.

Что даёт транспорт XHTTP

🔬 XHTTP — технология маскировки трафика

XHTTP — это современный транспорт для протокола VLESS, который передаёт данные через HTTP-запросы[reference:22]. В сочетании с Reality он обеспечивает:

  • Имитацию TLS-рукопожатия — соединение выглядит как стандартный HTTPS-запрос.
  • Маскировку под браузер Chrome — все параметры соответствуют реальному браузеру.
  • Отсутствие характерных сигнатур — DPI не может отличить ваш трафик от обычного веб-сёрфинга[reference:23].
  • Возможность использования CDN — XHTTP позволяет использовать CDN-серверы между клиентом и сервером[reference:24].

Благодаря этой технологии ваш VPN-трафик неотличим от обычного HTTPS. Даже самые продвинутые DPI-системы не смогут его заблокировать, потому что они видят обычный браузер Chrome, а не VPN-соединение.

Советы по безопасности

  • Используйте сложные пароли для панели и для SSH-доступа[reference:26].
  • Отключите вход по паролю для SSH и используйте ключи[reference:27].
  • Измените стандартный порт SSH на нестандартный[reference:28].
  • Регулярно обновляйте панель и Xray-core — скрипт установки уже настраивает автоматическое обновление[reference:29].
  • Включите двухфакторную аутентификацию для входа в панель, если это поддерживается[reference:30].
  • Используйте домен с валидным SSL-сертификатом для панели вместо IP-адреса[reference:31].
