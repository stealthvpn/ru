Внедрение систем обнаружения вторжений: Fail2Ban и CrowdSec

Любой сервер, выходящий в интернет, ежедневно подвергается тысячам попыток взлома — брутфорс SSH, сканирование портов, DDoS-атаки. Для защиты от таких угроз используются системы обнаружения вторжений (IDS/IPS). Два самых популярных инструмента с открытым исходным кодом — Fail2Ban и CrowdSec. В этой статье я расскажу, как их внедрить, сравню их возможности и покажу, как они работают в связке с личным VPN-сервером на VPS.

Что такое системы обнаружения вторжений?

IDS (Intrusion Detection System) — это программное обеспечение, которое анализирует сетевой трафик или логи системы на предмет подозрительной активности. IPS (Intrusion Prevention System) — это IDS, который не только обнаруживает, но и автоматически блокирует атаки. Fail2Ban и CrowdSec относятся к классу IPS, так как могут блокировать IP-адреса злоумышленников.

Основные угрозы, от которых защищают эти системы:

  • Брутфорс — перебор паролей (SSH, FTP, веб-панели).
  • Сканирование портов — разведка перед атакой.
  • DDoS-атаки — переполнение ресурсов сервера.
  • Вредоносные запросы — SQL-инъекции, XSS и другие веб-уязвимости.

Fail2Ban — классика защиты

Fail2Ban — это утилита, которая сканирует логи сервисов (например, /var/log/auth.log для SSH) и банит IP-адреса, с которых приходит слишком много неудачных попыток входа. Она появилась в 2004 году и до сих пор остаётся стандартом де-факто для защиты VPS.

Как работает Fail2Ban

Fail2Ban состоит из трёх основных компонентов:

  • Фильтры (filters) — регулярные выражения для поиска подозрительных строк в логах.
  • Действия (actions) — команды, которые выполняются при срабатывании фильтра (например, добавить IP в iptables).
  • Тюрьмы (jails) — связка фильтра и действия с настройками (количество попыток, время бана).

📝 Пример конфигурации тюрьмы для SSH:

[sshd]
enabled = true
port = ssh
filter = sshd
logpath = /var/log/auth.log
maxretry = 3
bantime = 3600
findtime = 600

Эта конфигурация банит IP на 1 час после 3 неудачных попыток входа за 10 минут.

Преимущества Fail2Ban

  • Простота — лёгкая установка и настройка.
  • Малые требования — работает на любом VPS с минимальными ресурсами.
  • Гибкость — можно создавать свои фильтры для любых приложений.
  • Бесплатно — open-source лицензия GPL.

Недостатки Fail2Ban

  • Реактивный подход — банит только после того, как атака уже началась.
  • Локальная база — не обменивается информацией с другими серверами.
  • Может не справляться с распределёнными атаками — если атакующие используют ботнет, каждый IP делает 1–2 попытки, и Fail2Ban их не замечает.

CrowdSec — современный коллективный интеллект

CrowdSec — это более новая система (вышла в 2020 году), которая работает по принципу «коллективного иммунитета». Она не только анализирует логи локального сервера, но и обменивается данными об атакующих IP с другими участниками сети CrowdSec. Если один сервер заметил атаку с IP, этот IP блокируется на всех серверах, подключённых к сети.

Ключевые особенности CrowdSec

  • Коллективная защита — более 100 000 участников по всему миру обмениваются данными об угрозах.
  • Поведенческий анализ — не только по логам, но и по аномалиям в трафике.
  • Гибкие сценарии (scenarios) — можно создавать сложные правила обнаружения.
  • Интеграция с фаерволами — поддерживает iptables, nftables, Cloudflare, и другие.
  • API и веб-интерфейс — удобное управление и мониторинг.

📊 Пример сценария CrowdSec для защиты от брутфорса SSH:

name: crowdsecurity/ssh-bf
description: "Detect SSH bruteforce"
filter: "evt.Meta.log_type == 'ssh_failed_auth'"
groupby: "evt.Meta.source_ip"
distinct: "evt.Meta.source_ip"
capacity: 5
leak: 30s
blackhole: 10m
action: "ban"

Этот сценарий банит IP на 10 минут после 5 неудачных попыток за 30 секунд.

Преимущества CrowdSec перед Fail2Ban

  • Превентивная защита — блокирует известные злонамеренные IP ещё до того, как они попытаются атаковать ваш сервер.
  • Распределённый интеллект — защита от распределённых атак (ботнетов).
  • Масштабируемость — подходит для больших инфраструктур.
  • Активное сообщество — постоянно обновляемые сценарии и базы данных угроз.

Недостатки CrowdSec

  • Более сложная настройка — требует изучения документации.
  • Потребляет больше ресурсов — особенно на старых серверах.
  • Зависит от внешнего API — для работы коллективной защиты требуется интернет-доступ.

Пошаговая установка и настройка

Установка Fail2Ban (на примере Ubuntu 22.04)

# Установка
sudo apt update
sudo apt install fail2ban -y

# Копирование базовой конфигурации
sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local

# Редактирование jail.local
sudo nano /etc/fail2ban/jail.local

# Включение тюрьмы для SSH
[sshd]
enabled = true
port = ssh
filter = sshd
logpath = /var/log/auth.log
maxretry = 3
bantime = 3600
findtime = 600

# Перезапуск службы
sudo systemctl restart fail2ban
sudo systemctl enable fail2ban

Установка CrowdSec

# Установка из официального репозитория
curl -s https://packagecloud.io/install/repositories/crowdsec/crowdsec/script.deb.sh | sudo bash
sudo apt install crowdsec -y

# Установка модуля для SSH
sudo apt install crowdsec-mod-security -y

# Запуск и активация
sudo systemctl start crowdsec
sudo systemctl enable crowdsec

# Проверка статуса
sudo cscli metrics
sudo cscli alerts list

После установки CrowdSec автоматически подключается к сети и начинает обмениваться данными. Вы можете настроить дополнительные сценарии через sudo cscli scenarios list.

Защита сервера и анонимность: почему это важно для VPN

Если вы используете VPS для личного VPN, безопасность сервера — это не просто «хорошо бы», а критическая необходимость. Взломанный сервер может стать частью ботнета, использоваться для атак на другие системы или для компрометации вашего трафика. Fail2Ban и CrowdSec — это первая линия обороны.

Но даже самый защищённый сервер бесполезен, если ваш VPN-трафик легко обнаружить и заблокировать с помощью DPI (Deep Packet Inspection). Именно здесь на помощь приходит XHTTP + Reality.

🚀 Почему личный VPN на VPS с XHTTP+Reality — это выбор профессионалов

В отличие от обычных VPN-сервисов, где 1000+ человек сидят на одном IP, вы получаете персональный VPS. Это даёт вам не только уникальный IP-адрес, но и полный контроль над сервером, включая настройку Fail2Ban или CrowdSec для максимальной защиты.

  • Персональный IP — вы не делите его с сотнями незнакомцев, что снижает риск попадания в бан-листы.
  • Технология XHTTP + Reality — ваш трафик маскируется под обычный HTTPS-запрос к легитимному сайту. DPI-системы видят браузер Chrome, а не VPN.
  • Имитация браузера Chrome — все параметры соединения (версия TLS, набор шифров, заголовки) соответствуют реальному браузеру, поэтому DPI не отличает ваш трафик от обычного веб-сёрфинга.
  • Нас в 1000 раз сложнее заблокировать и отследить — потому что каждый клиент — это отдельный выделенный сервер.
