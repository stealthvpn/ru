Настройка VPN на VPS с использованием PPTP: полное руководство

Несмотря на то, что сегодня он считается устаревшим и небезопасным, многие администраторы всё ещё используют его для быстрого развёртывания VPN-доступа. В конце статьи мы сравним PPTP с передовыми технологиями и покажем, почему личный VPN на VPS с XHTTP + Reality — это выбор будущего.

Что такое PPTP?

PPTP (Point-to-Point Tunneling Protocol) — это сетевой протокол, разработанный Microsoft в 1990-х годах для создания VPN-соединений. Он позволяет инкапсулировать сетевые пакеты в IP-пакеты и передавать их по интернету. PPTP широко использовался в Windows 95 и более поздних версиях, что сделало его одним из самых доступных протоколов.

📌 Ключевые особенности PPTP:

  • Простота настройки — минимальное количество параметров.
  • Встроенная поддержка — работает «из коробки» на Windows, macOS, Android и iOS.
  • Низкая производительность — не требует больших вычислительных ресурсов.
  • Слабое шифрование — использует MPPE (Microsoft Point-to-Point Encryption), которое было взломано.

Несмотря на свою популярность в прошлом, сегодня PPTP считается небезопасным. Его шифрование уязвимо для атак, а протокол легко детектируется системами DPI (Deep Packet Inspection). Однако для некоторых задач, например, для временного доступа к ресурсам без критичных данных, PPTP может быть полезен.

Подготовка VPS

Для настройки PPTP-сервера вам понадобится VPS (Virtual Private Server) с установленной операционной системой (мы будем использовать Ubuntu 20.04 LTS). Вы можете арендовать VPS у любого провайдера, но для максимальной безопасности и производительности мы рекомендуем использовать выделенный сервер от VPS-VPN.

Убедитесь, что ваш VPS имеет:

  • Статический публичный IP-адрес.
  • Доступ по SSH с правами root.
  • Открытые порты: 1723 (TCP) и протокол GRE (для PPTP).

Установка PPTP-сервера

Мы будем использовать пакет pptpd — популярную реализацию PPTP-сервера для Linux.

Шаг 1. Обновление системы и установка пакетов

sudo apt update && sudo apt upgrade -y sudo apt install pptpd -y

Шаг 2. Настройка конфигурации

Отредактируйте файл /etc/pptpd.conf:

sudo nano /etc/pptpd.conf

Добавьте или раскомментируйте следующие строки:

localip 192.168.100.1 remoteip 192.168.100.100-200

Здесь localip — IP-адрес сервера в VPN-сети, а remoteip — пул IP-адресов для клиентов.

Шаг 3. Настройка пользователей

Отредактируйте файл /etc/ppp/chap-secrets для добавления учётных записей:

sudo nano /etc/ppp/chap-secrets

Формат записи:

# client server secret IP addresses username1 pptpd password1 * username2 pptpd password2 *

Замените username1 и password1 на свои логин и пароль.

Шаг 4. Настройка DNS

Чтобы клиенты могли разрешать доменные имена, добавьте DNS-серверы в файл /etc/ppp/pptpd-options:

sudo nano /etc/ppp/pptpd-options

Добавьте строки:

ms-dns 8.8.8.8 ms-dns 1.1.1.1

Шаг 5. Перезапуск PPTP-сервера

sudo systemctl restart pptpd sudo systemctl enable pptpd

Настройка firewall

Чтобы клиенты могли подключаться, необходимо открыть порт 1723 (TCP) и разрешить протокол GRE. Если вы используете ufw, выполните команды:

sudo ufw allow 1723/tcp sudo ufw allow proto gre sudo ufw reload

Важно: Некоторые хостинг-провайдеры блокируют GRE по умолчанию. Если клиенты не могут подключиться, обратитесь в поддержку провайдера.

Подключение клиентов

Настройка клиента зависит от операционной системы. Рассмотрим самые популярные.

Windows

  1. Откройте «Параметры» → «Сеть и Интернет» → «VPN».
  2. Нажмите «Добавить VPN-подключение».
  3. Выберите PPTP в качестве типа VPN.
  4. Введите IP-адрес вашего VPS, логин и пароль.
  5. Подключитесь.

macOS

  1. Откройте «Системные настройки» → «Сеть».
  2. Нажмите «+» и выберите «VPN» с типом «PPTP».
  3. Введите IP-адрес VPS, имя пользователя и пароль.
  4. Нажмите «Подключиться».

Android / iOS

В настройках сети выберите «VPN» → «Добавить VPN» → тип «PPTP». Введите IP-адрес, логин и пароль.

«PPTP настраивается за 5 минут, но это решение из прошлого века. Используйте его только для некритичных задач и временного доступа».

Риски и ограничения PPTP

Несмотря на простоту, PPTP имеет серьёзные недостатки:

  • Уязвимость шифрования — MPPE можно взломать за несколько часов.
  • Отсутствие аутентификации сервера — возможны атаки «человек посередине».
  • Лёгкое обнаружение DPI — большинство провайдеров и систем фильтрации мгновенно блокируют PPTP.
  • Ограниченная пропускная способность — не подходит для потокового видео и больших объёмов данных.

Именно поэтому сегодня PPTP практически не используется в серьёзных проектах. На смену ему пришли современные протоколы, такие как WireGuard, OpenVPN и, особенно, XHTTP + Reality.
