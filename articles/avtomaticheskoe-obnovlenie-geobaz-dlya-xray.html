Автоматическое обновление геобаз для Xray: полное руководство

Xray — это мощный прокси-платформа, которая использует файлы геобаз (geoip.dat и geosite.dat) для маршрутизации трафика на основе географического положения и категорий сайтов. Регулярное обновление этих баз критически важно для корректной работы маршрутизации, обхода блокировок и обеспечения высокой производительности. В этой статье мы подробно разберём, как настроить автоматическое обновление геобаз для Xray, какие инструменты для этого использовать, и почему использование личного VPS-сервера с технологией XHTTP + Reality даёт вам максимальную защиту от DPI и блокировок.

Что такое геобазы и зачем их обновлять

Геобазы Xray — это два файла:

  • geoip.dat — содержит соответствия IP-адресов странам и регионам. Используется для маршрутизации трафика по географическому признаку (например, направить трафик в Россию через определённый прокси).
  • geosite.dat — содержит списки доменов, сгруппированные по категориям (например, google, youtube, telegram, bank и т.д.). Используется для маршрутизации на основе категорий сайтов.

Эти базы постоянно обновляются: появляются новые IP-адреса, меняются принадлежности доменов, добавляются новые категории. Если вы используете устаревшие геобазы, ваш Xray может:

  • Неправильно маршрутизировать трафик (например, отправлять российский трафик через зарубежный прокси).
  • Пропускать блокировки (новые домены могут не попадать в списки).
  • Снижать производительность из-за неоптимальной маршрутизации.

Поэтому регулярное обновление геобаз — это обязательная практика для администраторов Xray-серверов.

Ручное обновление геобаз

Самый простой способ обновить геобазы — скачать их вручную с официального репозитория v2fly/geoip и v2fly/domain-list-community (или использовать сборки от Loyalsoldier, которые часто более свежие).

wget -O /usr/local/share/xray/geoip.dat https://github.com/Loyalsoldier/geoip/releases/latest/download/geoip.dat
wget -O /usr/local/share/xray/geosite.dat https://github.com/Loyalsoldier/geoip/releases/latest/download/geosite.dat

После скачивания нужно перезапустить Xray для применения новых баз:

systemctl restart xray

Но такой подход требует ручного вмешательства каждый раз, когда вы хотите обновиться. Для production-серверов это неприемлемо.

Автоматизация через cron

Самый популярный способ автоматизации — использование cron — планировщика задач Linux. Создадим скрипт, который будет запускаться, например, каждый день в 3 часа ночи.

Создайте файл /usr/local/bin/update-geobases.sh:

#!/bin/bash
# Путь к директории с геобазами
GEO_DIR="/usr/local/share/xray"
# Временная директория для скачивания
TMP_DIR="/tmp/geobases"
mkdir -p "$TMP_DIR"
# Скачиваем новые файлы
wget -q -O "$TMP_DIR/geoip.dat" https://github.com/Loyalsoldier/geoip/releases/latest/download/geoip.dat
wget -q -O "$TMP_DIR/geosite.dat" https://github.com/Loyalsoldier/geoip/releases/latest/download/geosite.dat
# Проверяем, что файлы не пустые
if [ -s "$TMP_DIR/geoip.dat" ] && [ -s "$TMP_DIR/geosite.dat" ]; then
  cp "$TMP_DIR/geoip.dat" "$GEO_DIR/geoip.dat"
  cp "$TMP_DIR/geosite.dat" "$GEO_DIR/geosite.dat"
  systemctl restart xray
  echo "Геобазы обновлены $(date)" >> /var/log/geobases-update.log
else
  echo "Ошибка скачивания геобаз $(date)" >> /var/log/geobases-update.log
fi
rm -rf "$TMP_DIR"

Сделайте скрипт исполняемым:

chmod +x /usr/local/bin/update-geobases.sh

Добавьте задание в crontab (crontab -e):

0 3 * * * /usr/local/bin/update-geobases.sh

Теперь каждую ночь геобазы будут обновляться, а Xray — перезапускаться.

Использование systemd-таймеров

Если вы предпочитаете systemd, можно создать сервис и таймер. Это даёт более гибкое управление и логирование.

Создайте файл /etc/systemd/system/update-geobases.service:

[Unit]
Description=Update Xray geobases
After=network.target

[Service]
Type=oneshot
ExecStart=/usr/local/bin/update-geobases.sh
User=root

Затем создайте таймер /etc/systemd/system/update-geobases.timer:

[Unit]
Description=Timer for Xray geobases update
Requires=update-geobases.service

[Timer]
OnCalendar=daily
Persistent=true

[Install]
WantedBy=timers.target

Активируйте таймер:

systemctl daemon-reload
systemctl enable update-geobases.timer
systemctl start update-geobases.timer

Скрипт для обновления с проверкой

В приведённом выше скрипте есть проверка на пустые файлы. Это важно, чтобы избежать повреждения баз в случае сбоя сети. Можно добавить проверку контрольной суммы, чтобы обновлять только при изменении файлов. Например, использовать wget с опцией -N или сравнивать хеши. Но для большинства случаев достаточно проверки размера.

Также можно использовать утилиту v2ray-geodata из некоторых дистрибутивов, но она не всегда доступна.

Настройка Xray для использования обновлённых баз

Убедитесь, что в вашем конфигурационном файле Xray (/etc/xray/config.json) правильно указаны пути к геобазам. Обычно они находятся в разделе "routing":

{
  "routing": {
    "domainStrategy": "IPIfNonMatch",
    "rules": [
      {
        "type": "field",
        "outboundTag": "block",
        "domain": ["geosite:category-ads-all"]
      },
      {
        "type": "field",
        "outboundTag": "proxy",
        "domain": ["geosite:google", "geosite:youtube"]
      }
    ]
  }
}

Путь к файлам указывается в настройках "inbounds" или глобально. Обычно Xray ищет их в /usr/local/share/xray/ по умолчанию. Если вы изменили путь, укажите его явно через опцию "assetsPath" или переменную окружения XRAY_LOCATION_ASSETS.

Личный VPS vs массовые VPN

🚀 Почему личный VPS — лучшее решение для Xray

В отличие от обычных VPN-сервисов, где 1000+ человек сидят на одном IP, вы получаете персональный VPS.

XHTTP + Reality — защита от DPI

Даже если вы используете Xray с актуальными геобазами, ваш трафик может быть обнаружен системами DPI (Deep Packet Inspection) из-за характерных сигнатур протокола. XHTTP + Reality решает эту проблему, маскируя трафик под обычный HTTPS-запрос к легитимному сайту.

🔬 XHTTP + Reality: как это работает

Эта технология маскирует VPN-трафик под обычный HTTPS-запрос к легитимному сайту. DPI не может отличить ваше соединение от реального браузера Chrome, потому что:

  • Имитация TLS-рукопожатия — соединение выглядит как стандартный HTTPS-запрос.
  • Маскировка под Chrome — все параметры соответствуют реальному браузеру (версия, набор шифров, заголовки).
  • Отсутствие характерных сигнатур — в отличие от стандартного Xray или других протоколов, у XHTTP нет узнаваемых паттернов.
  • Динамическая адаптация — трафик меняется в зависимости от контекста.

Благодаря этой технологии ваш трафик неотличим от обычного веб-сёрфинга. Это делает XHTTP + Reality идеальным решением для стран с жёсткой фильтрацией интернета и активным DPI.

📌 Ключевое преимущество:

DPI-системы видят ваш трафик как обычный HTTPS-запрос к, например, google.com или cloudflare.com. Они не могут определить, что внутри этого запроса передаются VPN-данные. Нас в 1000 раз сложнее заблокировать и отследить.
