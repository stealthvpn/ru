Что такое 3x-UI и зачем он нужен

3x-UI — это продвинутая веб-панель управления для Xray-core, форк оригинального проекта X-UI[reference:2]. Она предоставляет интуитивно понятный графический интерфейс для настройки VPN-сервера, управления пользователями, мониторинга трафика и, самое главное, — для конфигурации XHTTP + Reality.

🚀 3x-UI

Протоколы: VLESS, VMess, Trojan, Shadowsocks, WireGuard

Маскировка: XHTTP + Reality (имитация Chrome)

Управление: Веб-панель, multi-user

★ Рекомендуемый

🔷 X-UI (старый)

Протоколы: VLESS, VMess, Trojan

Маскировка: Только TLS

Управление: Веб-панель

Устаревает

Ключевое преимущество 3x-UI перед другими панелями — это поддержка XHTTP транспорта в связке с Reality[reference:3]. XHTTP позволяет маскировать трафик под обычные HTTP/HTTPS-запросы, а Reality добавляет уровень имитации TLS-рукопожатия реального браузера. Вместе они создают соединение, которое DPI-системы не могут отличить от обычного веб-сёрфинга[reference:4].

Требования к VPS-серверу

Для установки 3x-UI вам понадобится VPS-сервер (Virtual Private Server) с минимальными характеристиками[reference:5]:

  • ОС: Ubuntu 20.04 или 22.04 (рекомендуется) / Debian 11+
  • CPU: 1 vCPU (минимально) / 2 vCPU (рекомендуется)
  • RAM: 1 ГБ (минимально) / 2 ГБ (рекомендуется)
  • Диск: 10 ГБ SSD
  • Доступ в интернет: Выделенный публичный IP-адрес

Главное отличие подхода VPS-VPN в том, что вы получаете персональный выделенный VPS, а не делите один IP-адрес с тысячами других пользователей, как в обычных VPN-сервисах[reference:6].

🔥 Почему личный VPS — это единственное правильное решение?

В отличие от обычных VPN, где 1000+ человек сидят на одном IP, вы получаете персональный VPS.

  • Никаких «шумных соседей»: ваш IP-адрес используете только вы. Никто не сможет «подставить» вас своей активностью.
  • Стабильная скорость: пропускная способность не делится между сотнями пользователей.
  • Минимальный риск блокировок: массовые IP-адреса часто попадают в бан-листы из-за действий других пользователей. Ваш личный IP — чистый.
  • Полный контроль: вы сами управляете сервером, выбираете протоколы и настройки безопасности.
  • Технология XHTTP + Reality: ваш трафик маскируется под обычный HTTPS, DPI не может его отличить от обычного браузера Chrome.

Установка 3x-UI — пошаговая инструкция

Мы будем использовать официальный скрипт установки от разработчика 3x-UI. Все команды выполняются от root-пользователя или через sudo.

ШАГ 1

Подключение к серверу

Подключитесь к вашему VPS по SSH:

ssh root@ваш_ip_сервера

Введите пароль root (или используйте SSH-ключ).

ШАГ 2

Обновление системы

Перед установкой обязательно обновите пакеты:

apt update && apt upgrade -y
ШАГ 3

Загрузка и запуск скрипта установки

Скачайте официальный установщик 3x-UI и запустите его[reference:8]:

bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)

Скрипт автоматически определит архитектуру вашего сервера и установит все необходимые компоненты[reference:9].

ШАГ 4

Настройка параметров установки

В процессе установки скрипт задаст несколько вопросов[reference:10]:

  • Порт панели: по умолчанию 2053 (рекомендуется оставить или изменить на свой)
  • Путь к панели: можно оставить по умолчанию или задать свой (например, /3xui)
  • Логин и пароль: задайте надёжные учётные данные для входа в веб-панель

Скрипт также может предложить настроить UFW (брандмауэр) — соглашайтесь, чтобы защитить сервер[reference:11].

ШАГ 5

Завершение установки

После успешной установки вы увидите сообщение с адресом панели, логином и паролем. Сохраните эти данные.

Панель будет доступна по адресу: https://ваш_ip_сервера:порт/путь

🔑 Важно:

По умолчанию используется самоподписанный SSL-сертификат. Браузер покажет предупреждение — это нормально. Для production-использования рекомендуется настроить Let's Encrypt[reference:12].

Первоначальная настройка панели

Зайдите в веб-панель 3x-UI по полученному адресу. Введите логин и пароль, которые вы задали при установке.

После входа вы увидите главную панель управления. Основные разделы:

  • Inbounds (Входящие подключения): здесь создаются и настраиваются VPN-подключения
  • Settings (Настройки): общие параметры панели и Xray-core
  • System Status (Статус системы): мониторинг нагрузки, трафика и подключений

⚙️ Рекомендация:

Сразу после входа смените пароль администратора в разделе Settings → Admin Settings. Также рекомендуется включить двухфакторную аутентификацию (2FA) для дополнительной безопасности.

Настройка XHTTP + Reality — главный шаг

Именно здесь создаётся соединение, которое DPI не может обнаружить. Мы настроим VLESS + XHTTP + Reality — самую современную и надёжную связку[reference:13].

ШАГ 1

Создание нового Inbound

В панели 3x-UI перейдите в раздел Inbounds и нажмите Add Inbound.

ШАГ 2

Выбор протокола и настройка порта

  • Protocol: выберите VLESS
  • Port: установите 443 (стандартный HTTPS-порт) — это критически важно для маскировки
  • Remark: дайте название, например, "My XHTTP Reality"
ШАГ 3

Настройка транспорта XHTTP

В разделе Transport выберите XHTTP[reference:14].

Настройте параметры XHTTP:

  • Path: укажите путь, например, /xhttp или любой другой[reference:15]
  • Host: можно оставить пустым или указать домен для маскировки
ШАГ 4

Настройка Reality

В разделе Reality Settings активируйте Reality и настройте следующие параметры[reference:16]:

  • Destination: укажите легитимный сайт для маскировки, например, www.google.com или www.cloudflare.com
  • Server Name (SNI): тот же домен, что и в Destination
  • Private Key / Public Key: сгенерируйте автоматически (кнопка Generate)
  • Short IDs: сгенерируйте или оставьте по умолчанию

🔬 Как это работает:

Reality имитирует TLS-рукопожатие с указанным сайтом (например, Google). DPI видит, что ваш сервер общается с Google по HTTPS, и пропускает трафик. Внутри этого защищённого туннеля передаётся ваш VPN-трафик[reference:17].

ШАГ 5

Настройка пользователя

В разделе Clients добавьте нового пользователя:

  • Email: укажите любой email (например, user@example.com)
  • Flow: для VLESS с XHTTP рекомендуется установить xtls-rprx-vision[reference:18]
  • Limit IPs: оставьте 0 (без ограничений) или укажите нужное количество
  • Expire time: оставьте пустым (без ограничения) или укажите дату

Нажмите Add, а затем Save для сохранения всего Inbound.

ШАГ 6

Получение ссылки для подключения

После сохранения Inbound нажмите на иконку QR-кода или Copy рядом с созданным пользователем. Вы получите VLESS-ссылку для импорта в клиентское приложение (например, v2rayN, Nekoray, Sing-box).

Ссылка будет выглядеть примерно так:

vless://uuid@ваш_ip:443?encryption=none&flow=xtls-rprx-vision&security=reality&sni=www.google.com&fp=chrome&pbk=публичный_ключ&sid=short_id&type=xhttp&path=/xhttp#My%20XHTTP%20Reality

Обратите внимание на параметры:

  • fp=chrome — имитация отпечатка браузера Chrome[reference:19]
  • type=xhttp — использование XHTTP транспорта
  • security=reality — включение Reality

Дополнительные настройки для максимальной защиты

Настройка маршрутизации (Routing)

В 3x-UI есть встроенные инструменты для настройки маршрутизации. Рекомендуется заблокировать IP-адреса из России и других стран с активной цензурой[reference:21]:

  • Перейдите в Xray Configs → Basics → Basic Routing
  • Добавьте опцию Russia в поле Block IPs
  • Добавьте другие страны при необходимости

Настройка DNS для предотвращения утечек

Утечки DNS — одна из главных проблем VPN. В 3x-UI вы можете настроить безопасные DNS-серверы:

  • Перейдите в Xray Configs → DNS
  • Укажите надёжные DNS, например, 1.1.1.1 (Cloudflare) или 8.8.8.8 (Google)
  • Включите опцию DNS over HTTPS (DoH) для дополнительной защиты

Автоматическое обновление Xray-core

3x-UI позволяет автоматически обновлять ядро Xray до последней версии:

  • Перейдите в Settings → Xray Settings
  • Включите Auto Update
  • Настройте периодичность проверки обновлений
