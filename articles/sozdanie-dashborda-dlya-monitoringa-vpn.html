Зачем нужен дашборд для мониторинга VPN

Многие пользователи VPN воспринимают его как «чёрный ящик»: включил — и работает. Но если вы используете личный VPN на VPS, у вас появляется уникальная возможность заглянуть внутрь этого ящика и увидеть, что именно происходит на вашем сервере.

🎯 Основные цели дашборда:

  • Контроль нагрузки: загрузка CPU, память, сетевой трафик — чтобы вовремя заметить аномалии.
  • Мониторинг подключений: количество активных сессий, география пользователей (если у вас мульти-пользовательский доступ).
  • Отслеживание качества: пинг, джиттер, потеря пакетов — чтобы убедиться, что соединение стабильно.
  • Безопасность: обнаружение необычных всплесков трафика или попыток несанкционированного доступа.
  • Прозрачность: вы видите, что ваш трафик действительно маскируется под обычный HTTPS (XHTTP + Reality), и DPI не может его распознать.

В отличие от массовых VPN-сервисов, где вы не имеете доступа к серверной инфраструктуре, на личном VPS вы владеете всеми данными и можете настроить мониторинг под свои нужды.

Эта связка позволяет собирать данные с вашего VPS, сохранять их в базе временных рядов и отображать в удобном интерфейсе. Всё это работает на вашем же сервере, что гарантирует конфиденциальность — никакие данные не покидают ваш VPS.

Установка и настройка

Разберём пошаговую установку на примере сервера под управлением Ubuntu 22.04. Все команды выполняются через SSH — доступ к вашему VPS.

📦 Шаг 1: Установка Prometheus

wget https://github.com/prometheus/prometheus/releases/download/v2.45.0/prometheus-2.45.0.linux-amd64.tar.gz
tar xvf prometheus-2.45.0.linux-amd64.tar.gz
sudo mv prometheus-2.45.0.linux-amd64 /opt/prometheus
cd /opt/prometheus
./prometheus --config.file=prometheus.yml &

Prometheus будет слушать порт 9090. Проверьте: curl http://localhost:9090.

📦 Шаг 2: Установка Grafana

sudo apt-get install -y software-properties-common
sudo add-apt-repository "deb https://packages.grafana.com/oss/deb stable main"
wget -q -O - https://packages.grafana.com/gpg.key | sudo apt-key add -
sudo apt-get update
sudo apt-get install grafana
sudo systemctl start grafana-server
sudo systemctl enable grafana-server

Grafana будет доступна по порту 3000. Логин/пароль по умолчанию: admin/admin.

📦 Шаг 3: Настройка Node Exporter (системные метрики)

wget https://github.com/prometheus/node_exporter/releases/download/v1.6.0/node_exporter-1.6.0.linux-amd64.tar.gz
tar xvf node_exporter-1.6.0.linux-amd64.tar.gz
sudo mv node_exporter-1.6.0.linux-amd64/node_exporter /usr/local/bin/
node_exporter &

Теперь добавьте node_exporter в prometheus.yml как цель (target).

После этих шагов у вас есть базовая система сбора метрик. Далее нужно настроить дополнительные экспортёры для VPN-специфичных метрик (например, количество подключений, трафик по протоколам).

Какие метрики собирать

Для полноценного дашборда мы собираем несколько групп метрик:

  • Системные: CPU, RAM, диск, сетевые интерфейсы (входящий/исходящий трафик).
  • VPN-специфичные: количество активных туннелей, тип используемого протокола (XHTTP, Reality), объём переданных данных.
  • Сетевые: задержки (ping) до внешних ресурсов, потеря пакетов.
  • Безопасность: логи ошибок, количество неудачных попыток подключения, подозрительные IP.

Если вы используете XHTTP + Reality на своём VPS, вы можете добавить метрики, которые показывают, насколько эффективно трафик маскируется под обычный HTTPS — например, долю успешных рукопожатий TLS.

Создание дашборда в Grafana

Переходим к самому интересному — созданию визуального интерфейса.

📌 Пошаговая инструкция:

  1. Добавьте источник данных Prometheus в Grafana: настройки → Data Sources → Add → Prometheus, URL: http://localhost:9090.
  2. Создайте новый дашборд через кнопку «+» в боковом меню.
  3. Добавьте панели:
    • График загрузки CPU — запрос 100 - (avg(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100).
    • График использования RAM — запрос 1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes).
    • Сетевой трафик — запросы rate(node_network_receive_bytes_total[5m]) и rate(node_network_transmit_bytes_total[5m]).
    • Количество активных VPN-сессий (зависит от вашей реализации VPN-сервиса).
    • Статус XHTTP + Reality — можно добавить кастомный экспортёр, который проверяет доступность и маскировку.
  4. Настройте алерты — например, если загрузка CPU превышает 80% или количество подключений падает до нуля, отправляйте уведомление в Telegram или по email.

Готовый дашборд можно экспортировать в JSON и использовать как шаблон для других серверов. Так вы сможете быстро развернуть мониторинг на любом новом VPS.

Безопасность дашборда

Важно помнить, что дашборд — это тоже точка входа на ваш сервер. Поэтому мы рекомендуем:

  • Использовать сильные пароли для Grafana и Prometheus.
  • Настроить HTTPS для Grafana (с помощью Let's Encrypt).
  • Ограничить доступ к портам 9090 (Prometheus) и 3000 (Grafana) с помощью фаервола, разрешив только ваш IP или используя VPN-туннель для доступа к дашборду.
  • Регулярно обновлять компоненты мониторинга.

Если вы используете XHTTP + Reality, ваш дашборд будет защищён тем же уровнем маскировки, что и весь трафик — злоумышленники не смогут определить, что за портами скрываются системы мониторинга.
