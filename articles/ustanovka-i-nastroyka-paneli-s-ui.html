Установка и настройка панели S‑UI на VPS: полное руководство

Что такое S‑UI и зачем он нужен

S‑UI — это современная веб-панель управления для прокси-серверов на базе Xray или Sing‑box. Она предоставляет графический интерфейс для:

  • Управления инбаундами (входящими подключениями) — VLESS, Trojan, Shadowsocks, XHTTP и другие.
  • Создания пользователей с индивидуальными ключами и лимитами трафика.
  • Мониторинга — графики нагрузки, статистика по пользователям, история подключений.
  • Настройки маскировки — поддержка XHTTP + Reality, имитация браузера Chrome.

Благодаря S‑UI вам не нужно править JSON-конфиги вручную — все настройки выполняются через удобный веб-интерфейс, что особенно ценно для новичков и администраторов, которым нужно быстро разворачивать VPN-серверы.

Требования к VPS

Для установки S‑UI вам понадобится:

  • VPS с операционной системой Ubuntu 20.04 / 22.04 / Debian 11 / 12 (рекомендуем Ubuntu 22.04).
  • Минимальные характеристики: 1 vCPU, 1 ГБ RAM, 10 ГБ SSD — этого достаточно для панели и нескольких пользователей.
  • Открытый порт 80/443 для веб-интерфейса (можно использовать любой порт).
  • Доступ по SSH к серверу с правами root или sudo.

Главное преимущество S‑UI — он лёгкий и не требует много ресурсов, поэтому подойдёт даже для недорогих VPS.

Пошаговая установка S‑UI

Мы будем использовать официальный скрипт установки от разработчиков S‑UI. Все команды выполняются от root или через sudo.

Шаг 1. Подключение к серверу

Подключитесь к вашему VPS по SSH:

ssh root@ваш_ip_сервера

Шаг 2. Обновление системы и установка зависимостей

apt update && apt upgrade -y
apt install curl wget unzip -y

Шаг 3. Загрузка и запуск скрипта установки

Скачайте и выполните установочный скрипт S‑UI:

bash <(curl -Ls https://raw.githubusercontent.com/FranzKafkaYu/s-ui/master/install.sh)

Скрипт автоматически установит S‑UI, Xray и все необходимые компоненты. В процессе установки вас попросят ввести:

  • Порт веб-панели (по умолчанию 8080, можно оставить или изменить).
  • Логин и пароль для входа в панель.

После завершения установки вы увидите сообщение с адресом панели и учётными данными. Сохраните их.

✅ Установка завершена!

Теперь панель S‑UI доступна по адресу: http://ваш_ip_сервера:порт (например, http://123.45.67.89:8080).

Первоначальная настройка панели

Зайдите в веб-интерфейс S‑UI, введите логин и пароль, которые вы задали при установке.

Основные разделы панели:

  • Inbounds — список входящих подключений. Здесь вы создаёте новые узлы с разными протоколами.
  • Users — управление пользователями: добавление, удаление, просмотр трафика.
  • Settings — глобальные настройки: порт панели, SSL-сертификаты, уведомления.
  • Dashboard — статистика в реальном времени.

Рекомендуем сразу сменить пароль администратора и включить HTTPS (через Let's Encrypt) для безопасного доступа к панели.

Настройка XHTTP + Reality в S‑UI

Технология XHTTP + Reality — это современный способ маскировки VPN-трафика под обычный HTTPS-запрос. Она имитирует браузер Chrome, и DPI-системы не могут отличить ваше соединение от обычного веб-сёрфинга. Настроить её в S‑UI очень просто.

Шаг 1. Создание нового Inbound

Перейдите в раздел Inbounds → Add Inbound.

  • Protocol — выберите XHTTP (или VLESS с настройками Reality).
  • Port — укажите порт, например, 443 (рекомендуется использовать 443 для имитации HTTPS).
  • Flow — выберите xtls-rprx-vision или reality.
  • Destination — укажите домен для маскировки (например, www.google.com).
  • Public Key / Private Key — сгенерируйте пару ключей Reality (кнопка Generate).
  • Fingerprint — выберите chrome или random.

В S‑UI есть встроенный генератор параметров Reality, что сильно упрощает настройку.

🔬 Как это работает:

Ваш клиент (например, Nekoray, Sing-box) подключается к серверу на порту 443. Рукопожатие TLS имитирует реальный браузер Chrome, а внутри передаётся зашифрованный VPN-трафик. DPI видит только обычный HTTPS-запрос к Google и не может заблокировать или деанонимизировать соединение.

Шаг 2. Создание пользователя для XHTTP

После создания Inbound перейдите в Users → Add User, выберите созданный Inbound и сгенерируйте ключ (UUID). Сохраните конфигурацию — она понадобится для настройки клиентского приложения.

📌 Важно:

Благодаря XHTTP + Reality ваш трафик становится неотличимым от обычного браузера Chrome. Нас в 1000 раз сложнее заблокировать и отследить, чем массовые VPN-сервисы с общими IP-адресами.

Управление пользователями и мониторинг

S‑UI позволяет легко добавлять новых пользователей, устанавливать лимиты трафика и срок действия. Вы можете просматривать статистику по каждому пользователю: сколько данных передано, когда было последнее подключение.

  • Добавление пользователя: укажите имя, выберите Inbound, сгенерируйте ключ.
  • Лимиты: можно установить общий лимит трафика (например, 100 ГБ) или временное ограничение.
  • Экспорт конфигураций: панель может генерировать QR-коды и ссылки для быстрого импорта в клиенты.

Такой уровень контроля особенно полезен, если вы предоставляете доступ к VPN друзьям или коллегам.
