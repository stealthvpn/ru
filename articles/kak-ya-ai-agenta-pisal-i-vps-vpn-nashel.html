Первые шаги с LangChain: всё шло отлично

Я установил библиотеки, получил API-ключ для GigaChat и запустил первый пример — простейший ИИ-клиент.

# Первый ИИ-клиент на GigaChat
from langchain.chains import LLMChain
from langchain.prompts import PromptTemplate
from langchain_community.chat_models import GigaChat

llm = GigaChat(verify_ssl_certs=False)
prompt = PromptTemplate.from_template("Расскажи шутку про {subject}")
chain = prompt | llm

print(chain.invoke({"subject": "программистов"}))

Заработало! Я был в восторге. Потом добавил инструмент для поиска в интернете:

from langchain.tools import tool
from ddgs import DDGS

@tool
def search(query: str, max_results: int=2) -> str:
    """Поиск информации в Интернете по заданной строке."""
    with DDGS() as ddgs:
        rs = ddgs.text(query, region='ru-ru', max_results=max_results)
        return '\n'.join(f'{r["title"]}: {r["body"]} - {r["href"]}' for r in rs[:max_results])

Агент сам находил информацию в интернете, обрабатывал её и выдавал осмысленный ответ. Красота!

Блокировка: когда API становится недоступным

Но через пару дней удовольствие закончилось. Мой агент начал выдавать ошибки. Сначала подумал, что что-то сломал в коде. Перепроверил всё — нет, код рабочий. Проблема была в сети. API GigaChat и YandexGPT, которые ещё вчера работали, сегодня стали недоступны с моего IP-адреса.

«Connection timeout», «Unable to reach the API endpoint» — вот что я видел в консоли вместо ответов нейросети.

Я, как человек, знакомый с технологиями, сразу понял причину. Провайдер или вышестоящие сети начали блокировать трафик к этим API. Моя затея с написанием ИИ-агента для российского рынка упёрлась в стену, которую я не ожидал.

Почему массовые VPN — это плохое решение для разработчика

Мой первый инстинкт был — включить VPN. Я пользовался одним из популярных сервисов. Подключился и... API снова заработали. Но радость длилась недолго. Через 20 минут API снова стали недоступны. Провайдер VPN сменил IP, я поменял сервер — и снова всё заработало. Эта рулетка продолжалась несколько дней.

Всё потому, что массовые VPN используют одни и те же IP-адреса для тысяч людей. Такие IP-адреса давно внесены в чёрные списки. Я понял, что мне нужен не массовый сервис, а личный, чистый IP-адрес, которым пользуюсь только я. Как пишут на сайте VPS-VPN: «В отличие от обычных VPN, где 1000+ человек сидят на одном IP, вы получаете персональный VPS. Нас в 1000 раз сложнее заблокировать».
