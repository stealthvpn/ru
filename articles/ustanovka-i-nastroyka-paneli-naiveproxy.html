Установка и настройка панели NaiveProxy: пошаговое руководство

NaiveProxy — это современный инструмент, который позволяет организовать прокси-сервер с максимальной степенью маскировки трафика. В этой статье мы подробно разберём, как установить и настроить панель управления NaiveProxy на вашем собственном VPS-сервере.

  • Имитацию браузера Chrome — все параметры TLS совпадают с реальным браузером.

Что такое NaiveProxy и XHTTP + Reality

NaiveProxy — это прокси-сервер, который использует современные транспортные протоколы для максимальной скрытности. В основе лежит технология XHTTP + Reality, которая маскирует трафик под обычный HTTPS-запрос к реальному сайту (например, google.com или cloudflare.com).

✦ Как это работает:

  • Клиент устанавливает TLS-соединение с сервером, полностью имитируя рукопожатие браузера Chrome.
  • Внутри защищённого канала передаются данные прокси-протокола.
  • DPI-система видит только обычный HTTPS-трафик и не может обнаружить прокси.
  • Серверная часть (панель) управляет пользователями, ключами и статистикой.

В отличие от классических VPN (OpenVPN, WireGuard), NaiveProxy не оставляет характерных сигнатур в пакетах, что делает его идеальным инструментом для стран с жёсткой интернет-фильтрацией.

Подготовка сервера

Для установки панели NaiveProxy вам понадобится VPS-сервер (мы рекомендуем использовать серверы вне РФ) с установленной операционной системой Ubuntu 22.04 LTS или Debian 11.

1. Обновите систему:

sudo apt update && sudo apt upgrade -y

2. Установите необходимые пакеты:

sudo apt install -y curl wget git build-essential nginx certbot python3-certbot-nginx

3. Настройте домен и SSL-сертификат:

Панель NaiveProxy работает через HTTPS, поэтому вам понадобится домен, направленный на IP вашего VPS, и валидный SSL-сертификат (мы используем Let's Encrypt).

certbot --nginx -d your-domain.com

После этих шагов ваш сервер готов к установке панели управления.

Установка панели управления

Мы будем использовать официальный скрипт установки панели NaiveProxy (репозиторий поддерживается сообществом). Выполните команды последовательно:

1. Скачайте установочный скрипт:

git clone https://github.com/naiveproxy/naiveproxy-panel.git

2. Перейдите в папку и запустите установку:

cd naiveproxy-panel && sudo bash install.sh

3. Во время установки вас попросят ввести:

  • Домен, на котором будет работать панель.
  • Путь к SSL-сертификатам (обычно /etc/letsencrypt/live/your-domain.com/).
  • Логин и пароль для административной панели.

После завершения установки панель будет доступна по адресу https://your-domain.com/admin.

Настройка XHTTP + Reality

Теперь перейдём к самой важной части — настройке транспортного протокола XHTTP с Reality. Это гарантирует, что ваш трафик будет неотличим от обычного браузера.

1. В панели управления перейдите в раздел "Настройки" → "Транспорт".

Выберите тип XHTTP и укажите следующие параметры:

  • Reality: Включено (On).
  • Target site: Укажите легитимный сайт для маскировки (например, cloudflare.com).
  • Fingerprint: Выберите chrome для полной имитации браузера.
  • Public key / Private key: Автоматически сгенерируются панелью.

2. Настройте маскировку HTTP-заголовков:

В разделе "Headers" добавьте стандартные заголовки браузера Chrome:

User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/115.0.0.0 Safari/537.36
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,*/*;q=0.8

3. Сохраните настройки и перезапустите сервис:

sudo systemctl restart naiveproxy

✅ Готово! Теперь ваш NaiveProxy полностью настроен на работу с XHTTP + Reality.

DPI-системы будут видеть обычный HTTPS-трафик к указанному целевому сайту. Нас в 1000 раз сложнее заблокировать и отследить.

Подключение клиентов

Для подключения к вашему NaiveProxy-серверу используйте любой клиент, поддерживающий протокол NaiveProxy (например, NekoBox, SagerNet, или официальный клиент NaiveProxy).

1. В панели администратора создайте нового пользователя.

Укажите желаемый объём трафика (если нужно) и срок действия.

2. Скопируйте ссылку для подключения.

Панель сгенерирует URL вида:

naive+https://user:pass@your-domain.com:443?padding=true

3. Импортируйте ссылку в клиентское приложение.

Всё остальное клиент настроит автоматически.

Важно: все клиенты должны поддерживать XHTTP и Reality (это есть в актуальных версиях NekoBox, Nekoray, и др.).

Частые вопросы

❓ Можно ли использовать NaiveProxy на обычном хостинге?

Нет, для работы требуется VPS с доступом к настройкам ядра и сетевым стеком. Общий хостинг не подходит.

❓ Чем NaiveProxy отличается от Shadowsocks?

Shadowsocks имеет характерные сигнатуры DPI, а NaiveProxy с XHTTP + Reality полностью маскирует трафик под HTTPS браузера Chrome.

❓ Нужно ли покупать домен?

Да, для корректной работы TLS и Reality необходим домен, направленный на ваш VPS.

❓ Как быстро можно настроить?

С нашим подготовленным решением — менее 5 минут. Самостоятельная установка займёт около 30–40 минут.
