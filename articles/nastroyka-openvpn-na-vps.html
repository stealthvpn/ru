Подготовка VPS

Для начала вам понадобится виртуальный частный сервер (VPS). Мы рекомендуем выбирать серверы вне РФ, с хорошей пропускной способностью и поддержкой Ubuntu или Debian. В нашем примере будем использовать Ubuntu 22.04 LTS.

Шаг 1.
Подключитесь к серверу по SSH
$ ssh root@your-server-ip

Замените your-server-ip на реальный IP вашего VPS.

Шаг 2.
Обновите систему
# apt update && apt upgrade -y
Шаг 3.
Установите необходимые пакеты
# apt install curl wget ufw -y

Настроим базовый файрвол позже.

Установка OpenVPN

Мы будем использовать официальный скрипт установки OpenVPN, который автоматически настроит сервер и сгенерирует сертификаты.

Шаг 4.
Загрузите и запустите скрипт установки
# wget https://git.io/vpn -O openvpn-install.sh
# chmod +x openvpn-install.sh
# ./openvpn-install.sh

Скрипт задаст несколько вопросов. Рекомендуемые ответы:

  • IP адрес — оставьте автоматически определённый (или введите свой публичный IP).
  • Протокол — выберите UDP (быстрее) или TCP (надёжнее).
  • Порт — по умолчанию 1194.
  • DNS — можно оставить Google DNS (8.8.8.8).
  • Имя клиента — укажите имя для первого клиента, например client1.

После завершения скрипта вы получите файл client1.ovpn — это конфигурация для подключения.

Генерация сертификатов (если не использовали скрипт)

Если вы предпочитаете ручную настройку, то после установки OpenVPN (apt install openvpn easy-rsa) нужно сгенерировать сертификаты.

# cd /etc/openvpn/easy-rsa/
# ./easyrsa init-pki
# ./easyrsa build-ca nopass
# ./easyrsa gen-req server nopass
# ./easyrsa sign-req server server
# ./easyrsa gen-dh
# openvpn --genkey --secret ta.key

Однако для большинства пользователей автоматический скрипт — самый быстрый и безопасный путь.

Настройка сервера OpenVPN

Скрипт уже создал конфигурационный файл /etc/openvpn/server.conf. Если вы настраивали вручную, вот минимальная рабочая конфигурация:

port 1194
proto udp
dev tun
ca /etc/openvpn/easy-rsa/pki/ca.crt
cert /etc/openvpn/easy-rsa/pki/issued/server.crt
key /etc/openvpn/easy-rsa/pki/private/server.key
dh /etc/openvpn/easy-rsa/pki/dh.pem
server 10.8.0.0 255.255.255.0
push "redirect-gateway def1 bypass-dhcp"
push "dhcp-option DNS 8.8.8.8"
push "dhcp-option DNS 1.1.1.1"
keepalive 10 120
cipher AES-256-GCM
user nobody
group nogroup
persist-key
persist-tun
status openvpn-status.log
verb 3
Шаг 5.
Включите маршрутизацию и настройте файрвол
# echo 'net.ipv4.ip_forward=1' >> /etc/sysctl.conf
# sysctl -p
# ufw allow 1194/udp
# ufw allow OpenSSH
# ufw enable

И перезапустите OpenVPN:

# systemctl restart openvpn@server

Настройка клиента

Файл client1.ovpn, сгенерированный скриптом, уже содержит все необходимые настройки. Вам нужно:

  1. Скачать этот файл на своё устройство.
  2. Установить клиент OpenVPN (для Windows, macOS, Linux, Android, iOS).
  3. Импортировать конфигурацию и подключиться.

Если вы создаёте конфигурацию вручную, используйте примерно такой шаблон:

client
dev tun
proto udp
remote your-server-ip 1194
resolv-retry infinite
nobind
persist-key
persist-tun
remote-cert-tls server
cipher AES-256-GCM
verb 3
<ca>
-----BEGIN CERTIFICATE-----
... (содержимое ca.crt) ...
-----END CERTIFICATE-----
</ca>
<cert>
... (содержимое client.crt) ...
</cert>
<key>
... (содержимое client.key) ...
</key>
key-direction 1
<tls-auth>
... (содержимое ta.key) ...
</tls-auth>

Подключение и проверка

После импорта конфигурации в клиент просто нажмите «Подключиться». Если всё сделано правильно, вы получите IP-адрес из пула 10.8.0.0/24.

Проверьте, что ваш реальный IP изменился, перейдя на сайт 2ip.ru или whoer.net. Также можно проверить доступность ресурсов:

$ ping google.com

Если пинг идёт, значит, OpenVPN работает корректно.

XHTTP + Reality — максимальная защита от DPI

OpenVPN — отличный протокол, но у него есть один недостаток: DPI-системы научились распознавать его характерные сигнатуры. В странах с глубокой фильтрацией трафика OpenVPN может быть заблокирован. Именно здесь на помощь приходит технология XHTTP + Reality.

🔬 XHTTP + Reality: как это работает

Эта технология маскирует ваш VPN-трафик под обычный HTTPS-запрос к легитимному сайту. DPI не может отличить ваше соединение от реального браузера Chrome, потому что:

  • Имитация TLS-рукопожатия — соединение выглядит как стандартный HTTPS-запрос.
  • Маскировка под Chrome — все параметры соответствуют реальному браузеру (версия, набор шифров, заголовки).
  • Отсутствие характерных сигнатур — в отличие от OpenVPN или WireGuard, у XHTTP нет узнаваемых паттернов.
  • Динамическая адаптация — трафик меняется в зависимости от контекста.

📌 Ключевое преимущество:

DPI-системы видят ваш трафик как обычный HTTPS-запрос к, например, google.com или cloudflare.com. Они не могут определить, что внутри этого запроса передаются VPN-данные. Нас в 1000 раз сложнее заблокировать и отследить.
