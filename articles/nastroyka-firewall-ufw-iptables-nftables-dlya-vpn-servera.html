Настройка Firewall (UFW, iptables, nftables) для VPN-сервера

Межсетевой экран (firewall) — это первая линия обороны вашего VPN-сервера. Без правильно настроенного файрвола ваш VPS остаётся уязвимым для атак, несанкционированного доступа и DDoS. В этой статье мы разберём настройку трёх основных инструментов для управления трафиком на Linux-сервере: UFW (Uncomplicated Firewall), iptables и nftables. Вы узнаете, как открыть нужные порты, настроить NAT, разрешить IP-форвардинг и защитить свой VPN-сервер от внешних угроз.

Зачем нужен firewall для VPN

VPN-сервер — это шлюз между вами и интернетом. Если не настроить файрвол, злоумышленники могут:

  • Получить доступ к SSH — через стандартный порт 22 могут пытаться подобрать пароль.
  • Использовать ваш сервер для атак — открытые порты могут стать точкой входа для ботнетов.
  • Перехватывать трафик — без правильных правил NAT ваш VPN-трафик может утекать наружу.

Правильно настроенный firewall решает все эти проблемы: он закрывает все порты, кроме необходимых, разрешает IP-форвардинг и маскирует исходящий трафик через NAT.

Настройка UFW (Uncomplicated Firewall)

UFW — это самый простой способ управлять файрволом в Ubuntu и Debian. Он предоставляет интуитивно понятный интерфейс поверх iptables. Идеальный выбор для начинающих администраторов.

🔧 Базовая настройка UFW для VPN-сервера:

  1. Включение UFW: sudo ufw enable
  2. Разрешение SSH (важно!): sudo ufw allow 22/tcp (или ваш кастомный порт)
  3. Открытие порта для VPN:
    • Для OpenVPN (UDP 1194): sudo ufw allow 1194/udp
    • Для WireGuard (UDP 51820): sudo ufw allow 51820/udp
    • Для XHTTP + Reality (обычно 443/tcp): sudo ufw allow 443/tcp
  4. Разрешение IP-форвардинга:
    • Раскомментируйте net.ipv4.ip_forward=1 в /etc/sysctl.conf
    • Примените: sudo sysctl -p
  5. Настройка NAT (маскарадинг):
    • Добавьте в /etc/ufw/before.rules перед секцией *filter:
    *nat :POSTROUTING ACCEPT [0:0] -A POSTROUTING -s 10.8.0.0/24 -o eth0 -j MASQUERADE COMMIT
  6. Проверка статуса: sudo ufw status verbose

UFW автоматически применяет правила при перезагрузке. Это надёжное и простое решение для большинства сценариев[reference:0][reference:1].

Настройка iptables

iptables — классический инструмент для управления пакетным фильтром в Linux. Он даёт полный контроль над трафиком, но требует больше знаний. Используется в большинстве дистрибутивов по умолчанию.

🔧 Основные правила iptables для VPN:

  1. Политика по умолчанию (запрет всего): sudo iptables -P INPUT DROP sudo iptables -P FORWARD DROP sudo iptables -P OUTPUT ACCEPT
  2. Разрешить локальный трафик: sudo iptables -A INPUT -i lo -j ACCEPT
  3. Разрешить установленные соединения: sudo iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT
  4. Разрешить SSH: sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT
  5. Разрешить VPN-порт:
    • OpenVPN (UDP 1194): sudo iptables -A INPUT -p udp --dport 1194 -j ACCEPT
    • WireGuard (UDP 51820): sudo iptables -A INPUT -p udp --dport 51820 -j ACCEPT
  6. Разрешить IP-форвардинг для VPN-интерфейса: sudo iptables -A FORWARD -i tun+ -j ACCEPT sudo iptables -A FORWARD -o tun+ -j ACCEPT sudo iptables -A FORWARD -i wg0 -j ACCEPT
  7. Настройка NAT (маскарадинг): sudo iptables -t nat -A POSTROUTING -s 10.8.0.0/24 -o eth0 -j MASQUERADE
  8. Сохранение правил: sudo iptables-save > /etc/iptables/rules.v4 sudo ip6tables-save > /etc/iptables/rules.v6

iptables даёт максимальную гибкость. Например, можно использовать DNAT + MASQUERADE для создания прозрачного прокси-моста между клиентом и целевым сервером[reference:2]. Это полезно для скрытия реального IP вашего VPN-сервера.

Настройка nftables

nftables — это современная замена iptables. Он работает быстрее, имеет более удобный синтаксис и объединяет IPv4 и IPv6 в одном наборе правил. Начиная с 2023 года, nftables используется по умолчанию в новых установках многих дистрибутивов[reference:3].

🔧 Пример настройки nftables для VPN:

  1. Создание таблицы: sudo nft add table inet filter
  2. Создание цепочек: sudo nft add chain inet filter input { type filter hook input priority 0 \; policy drop \; } sudo nft add chain inet filter forward { type filter hook forward priority 0 \; policy drop \; } sudo nft add chain inet filter output { type filter hook output priority 0 \; policy accept \; }
  3. Разрешить локальный трафик и установленные соединения: sudo nft add rule inet filter input iif lo accept sudo nft add rule inet filter input ct state established,related accept
  4. Разрешить SSH и VPN: sudo nft add rule inet filter input tcp dport 22 accept sudo nft add rule inet filter input udp dport 1194 accept sudo nft add rule inet filter input udp dport 51820 accept sudo nft add rule inet filter input tcp dport 443 accept
  5. Разрешить IP-форвардинг для VPN: sudo nft add rule inet filter forward iif tun+ accept sudo nft add rule inet filter forward oif tun+ accept sudo nft add rule inet filter forward iif wg0 accept
  6. Настройка NAT (маскарадинг): sudo nft add table inet nat sudo nft add chain inet nat postrouting { type nat hook postrouting priority 100 \; } sudo nft add rule inet nat postrouting oif eth0 masquerade
  7. Сохранение правил: sudo nft list ruleset > /etc/nftables.conf

nftables поддерживает wildcards в именах интерфейсов, например tun* или wg*, что упрощает управление[reference:4]. Это особенно удобно, если у вас несколько VPN-туннелей.

Личный VPS vs Массовый VPN: почему firewall — не главное

В отличие от обычных VPN-сервисов, где 1000+ человек сидят на одном IP, вы получаете персональный VPS.

  • Никаких «шумных соседей»: ваш IP-адрес используете только вы. Никто не сможет «подставить» вас своей активностью.
  • Стабильная скорость: пропускная способность не делится между сотнями пользователей.
  • Минимальный риск блокировок: массовые IP-адреса часто попадают в бан-листы из-за действий других пользователей. Ваш личный IP — чистый.

XHTTP + Reality — защита от DPI

Современные DPI (Deep Packet Inspection) системы научились распознавать не только протоколы, но и характерные паттерны VPN-трафика. OpenVPN, WireGuard — все они имеют узнаваемые сигнатуры, по которым их можно заблокировать[reference:6]. Именно поэтому на сцену вышла технология XHTTP + Reality.

🔬 XHTTP + Reality: как это работает

Эта технология маскирует VPN-трафик под обычный HTTPS-запрос к легитимному сайту. DPI не может отличить ваше соединение от реального браузера Chrome, потому что:

  • Имитация TLS-рукопожатия — соединение выглядит как стандартный HTTPS-запрос[reference:7].
  • Маскировка под Chrome — все параметры соответствуют реальному браузеру (версия, набор шифров, заголовки)[reference:8].
  • Отсутствие характерных сигнатур — в отличие от OpenVPN или WireGuard, у XHTTP нет узнаваемых паттернов.
  • Динамическая адаптация — трафик меняется в зависимости от контекста[reference:9].

Благодаря этой технологии ваш VPN-трафик неотличим от обычного веб-сёрфинга. Это делает XHTTP + Reality идеальным решением для стран с жёсткой фильтрацией интернета и активным DPI.

📌 Ключевое преимущество:

DPI-системы видят ваш трафик как обычный HTTPS-запрос к, например, google.com или cloudflare.com. Они не могут определить, что внутри этого запроса передаются VPN-данные. Нас в 1000 раз сложнее заблокировать и отследить.

Заключение
