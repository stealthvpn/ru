Что такое Marzban и зачем он нужен

Marzban — это панель управления с открытым исходным кодом для Xray-core, которая предоставляет веб-интерфейс для администрирования VPN-сервера. Она поддерживает протоколы VMess, VLess, Trojan, Shadowsocks и, что самое важное, — XHTTP + Reality — технологию маскировки трафика под обычный HTTPS, которая делает ваш VPN-трафик неотличимым от обычного веб-сёрфинга в браузере Chrome.

Зачем нужна панель? Без неё вам пришлось бы вручную редактировать конфигурационные файлы Xray, создавать пользователей, следить за трафиком — это сложно и неудобно. Marzban автоматизирует все эти процессы и даёт удобный интерфейс, который можно использовать даже без глубоких знаний Linux.

🎯 Ключевые возможности Marzban:

  • Управление пользователями — создание, удаление, ограничение трафика и времени.
  • Поддержка всех современных протоколов — VMess, VLess, Trojan, Shadowsocks, XHTTP.
  • Интеграция с XHTTP + Reality — маскировка под браузер Chrome, обход DPI.
  • Telegram-бот — управление панелью через бота, выдача ссылок пользователям.
  • Мультиязычный интерфейс — доступен на русском, английском и других языках.

Требования к серверу

Для установки Marzban вам понадобится VPS-сервер (виртуальный выделенный сервер) с минимальными характеристиками:

  • ОС: Ubuntu 20.04 / 22.04 LTS или Debian 11 / 12 (рекомендуется Ubuntu 22.04).
  • CPU: 1 ядро (2+ ядра для высокой нагрузки).
  • RAM: от 1 ГБ (рекомендуется 2 ГБ и выше).
  • Диск: от 20 ГБ SSD.
  • Доступ в интернет: открытый порт 443 (для XHTTP+Reality) и 22 (SSH).
  • Домен (опционально): для XHTTP+Reality рекомендуется иметь собственный домен или использовать поддомен.

Главное преимущество — вы используете личный VPS, а не общий IP с сотнями других пользователей. Это кардинально повышает стабильность и снижает риск блокировок.

Установка панели Marzban

Установка Marzban максимально проста благодаря официальному скрипту. Выполните следующие команды на своём VPS (подключившись по SSH):

# Обновление системы
sudo apt update && sudo apt upgrade -y

# Установка Docker и Docker Compose
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER

# Установка Marzban
sudo bash -c "$(curl -sL https://github.com/Gozargah/Marzban/raw/master/install.sh)"

# Перезагрузите сессию или выполните newgrp docker

После установки панель будет доступна по адресу http://ваш-ip:8000. По умолчанию логин и пароль — admin / admin (обязательно смените их при первом входе).

Первоначальная настройка

После входа в веб-интерфейс выполните базовую настройку:

  1. Смените пароль администратора в разделе "Настройки" → "Администраторы".
  2. Настройте домен (если есть) — укажите его в разделе "Настройки" → "Основные". Это необходимо для корректной работы XHTTP+Reality.
  3. Создайте первого пользователя — в разделе "Пользователи" нажмите "Создать", укажите имя, лимит трафика и срок действия.
  4. Настройте уведомления — подключите Telegram-бота для автоматической выдачи ссылок и уведомлений об истечении срока.

На этом базовая настройка завершена. Теперь перейдём к самому важному — подключению XHTTP + Reality.

Настройка XHTTP + Reality в Marzban

Технология XHTTP + Reality — это эволюционный шаг в маскировке VPN-трафика. Она имитирует поведение браузера Chrome при установке HTTPS-соединения, поэтому DPI-системы не могут отличить ваш VPN-трафик от обычного веб-сёрфинга.

🔐 Как настроить XHTTP + Reality в Marzban:

  1. В веб-интерфейсе перейдите в раздел "Узлы".
  2. Нажмите "Создать узел".
  3. Выберите протокол VLess (XHTTP работает поверх VLess).
  4. В поле "Настройки потока" укажите xtls-rprx-vision — это требуется для Reality.
  5. Включите опцию "Reality" и укажите "target" — например, www.google.com:443 (или любой другой легитимный сайт).
  6. Задайте "serverNames" — список доменов, которые будет имитировать ваш сервер.
  7. Настройте "fingerprint" — выберите chrome для максимальной схожести с браузером Chrome.
  8. Сохраните узел и создайте пользователя с этим узлом.

После настройки ваш клиент (например, Nekoray, v2rayN или другой совместимый) получит конфигурацию, которая будет выглядеть как обычный HTTPS-запрос к Google или Cloudflare. DPI не сможет его заблокировать, потому что трафик неотличим от реального браузера.

"XHTTP + Reality — это не просто очередной протокол. Это полноценная имитация браузера, которая делает ваш VPN-трафик полностью невидимым для систем глубокого анализа пакетов." — из документации Xray.
